## 2. Взаимосвязи файлов

### 2.1 Основные файлы системы
```
index.html → app.js (использует JavaScript-функции)
index.html → styles.css (использует стили)
app.js → coop_data.json (сохраняет и загружает данные)
app.js → demo_data.json (загружает начальные данные)
app.js → C:\КООПЕРАНТ (через File System API)
```

### 2.2 Документация и руководства
```
README.md → Техническая_спецификация_информационной_системы_учета_пайщиков_кооператива.md
user_manual.md → data_structure.md
deployment_guide.md → backup_instructions.md
error_guide.md → all_functions.md
testing_plan.md → detailed_testing.md
security_policy.md → security_functions.md
training_program.md → usage_scenarios.md
```

### 2.3 Файлы данных
```
coop_data.json ← app.js (генерируется при работе)
demo_data.json → app.js (используется для начальной загрузки)
```

### 2.4 Вспомогательные скрипты
```
start.bat → index.html (открывает систему)
integrity_check.bat → все основные файлы (проверяет целостность)
```

## 4. Структура данных

### 4.1 Основные сущности
- **Пайщики:** members (массив объектов)
- **Паевые взносы:** payments (массив объектов)
- **Бухгалтерские проводки:** transactions (массив объектов)
- **Документы:** documents (массив объектов)
- **Заявления:** applications (массив объектов)
- **Заседания:** meetings (массив объектов)
- **Удостоверения:** certificates (массив объектов)

### 4.2 Связи между сущностями
```
members ↔ payments (один ко многим, через memberId)
payments ↔ transactions (один к одному/многим, через relatedPaymentId)
applications → members (один к одному при одобрении)
meetings ↔ decisions (один ко многим)
documents ↔ members (через метаданные)
```

## 5. Архитектурные зависимости

### 5.1 Зависимости модулей
```
Модуль пайщиков → Модуль паевых взносов (через связи memberId)
Модуль паевых взносов → Бухгалтерский модуль (через автоматические проводки)
Модуль заявлений → Модуль пайщиков (при одобрении создает пайщика)
Модуль заседаний → Модуль заявлений (для рассмотрения заявлений)
Модуль отчетов → Все другие модули (для формирования отчетов)
```

### 5.2 Зависимости файлов
```
app.js ← index.html (основной код)
styles.css ← index.html (стили интерфейса)
app.js ← File System API (для сохранения данных)
app.js ← coop_data.json (для хранения данных)
```

## 10. Заключение

Все созданные файлы образуют комплексное решение для автоматизации учета пайщиков потребительской кооперации. Структура файлов обеспечивает:
- Четкое разделение ответственности
- Легкость сопровождения
- Расширяемость функциональности
- Полное покрытие документацией
- Безопасность и надежность данных
- Соответствие требованиям законодательства

Система полностью готова к использованию и может быть адаптирована под конкретные потребности различных типов кооперативов.
