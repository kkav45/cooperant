# Каталог таблиц и форм системы

## 1. Основные таблицы

### 1.1 Таблица пайщиков (members-table)
- **Расположение:** index.html, раздел "Пайщики"
- **Назначение:** Отображение списка всех пайщиков кооператива
- **Колонки:**
  - ID: Уникальный идентификатор пайщика
  - ФИО: Полное имя пайщика
  - Дата вступления: Дата вступления в кооператив
  - Статус: Текущий статус пайщика (активный, кандидат, исключен и др.)
  - Контакт: Контактная информация
  - Действия: Кнопки управления (просмотр, изменить, выбытие, удалить)
- **Функции:** Поиск, фильтрация, добавление, редактирование, удаление
- **Источник данных:** Массив members в app.js

### 1.2 Таблица паевых взносов (payments-table)
- **Расположение:** index.html, раздел "Паевые взносы"
- **Назначение:** Отображение всех паевых взносов пайщиков
- **Колонки:**
  - ID: Уникальный идентификатор взноса
  - Пайщик: Имя связанного пайщика
  - Тип взноса: Вид паевого взноса (вступительный, паевой и др.)
  - Форма оплаты: Наличные, безналичные, имущество
  - Сумма: Размер взноса или статус "Не оценено" для имущества
  - Дата: Дата внесения взноса
  - Статус: Оплачено/Не оплачено/Ожидаемый
  - Действия: Кнопки управления (просмотр, изменить, оценить, удалить)
- **Функции:** Добавление, редактирование, удаление, оценка имущества
- **Источник данных:** Массив payments в app.js

### 1.3 Таблица бухгалтерских проводок (transactions-table)
- **Расположение:** index.html, раздел "Бухгалтерия"
- **Назначение:** Отображение всех бухгалтерских проводок
- **Колонки:**
  - ID: Уникальный идентификатор проводки
  - Дата: Дата проведения операции
  - Дебет: Дебетовый счет
  - Кредит: Кредитовый счет
  - Сумма: Сумма проводки
  - Описание: Описание операции
  - Действия: Кнопки управления (просмотр, изменить, удалить)
- **Функции:** Добавление, редактирование, удаление проводок
- **Источник данных:** Массив transactions в app.js

### 1.4 Таблица документов (documents-table)
- **Расположение:** index.html, раздел "Документы"
- **Назначение:** Отображение всех документов кооператива
- **Колонки:**
  - ID: Уникальный идентификатор документа
  - Название: Наименование документа
  - Тип: Категория документа (договор, отчет, платежный документ и др.)
  - Дата: Дата документа
  - Размер: Размер файла
  - Действия: Кнопки управления (просмотр, удалить)
- **Функции:** Загрузка, просмотр, удаление документов
- **Источник данных:** Массив documents в app.js

### 1.5 Таблица заявлений (applications-table)
- **Расположение:** index.html, раздел "Заявления"
- **Назначение:** Отображение всех заявлений на вступление
- **Колонки:**
  - ID: Уникальный идентификатор заявления
  - ФИО заявителя: Полное имя заявителя
  - Контакт: Контактная информация заявителя
  - Дата подачи: Дата подачи заявления
  - Статус: Статус обработки (на рассмотрении, одобрено, отклонено)
  - Действия: Кнопки управления (просмотр, обработать)
- **Функции:** Подача, просмотр, обработка заявлений
- **Источник данных:** Массив applications в app.js

### 1.6 Таблица заседаний (meetings-table)
- **Расположение:** index.html, раздел "Заседания"
- **Назначение:** Отображение всех заседаний кооператива
- **Колонки:**
  - ID: Уникальный идентификатор заседания
  - Дата: Дата проведения заседания
  - Тип: Вид заседания (правление, общее собрание и др.)
  - Тема: Тема заседания
  - Статус: Статус заседания (запланировано, проведено, отменено)
  - Действия: Кнопки управления (протокол, провести)
- **Функции:** Назначение, проведение, формирование протоколов
- **Источник данных:** Массив meetings в app.js

## 2. Основные формы системы

### 2.1 Форма добавления пайщика
- **Расположение:** Модальное окно в разделе "Пайщики"
- **Назначение:** Добавление нового пайщика в систему
- **Поля:**
  - ФИО (обязательное)
  - Статус (активный, кандидат и др.)
  - Дата вступления (по умолчанию сегодня)
  - Контакт (телефон, email)
  - Адрес
  - Примечания
- **Валидация:** Проверка обязательных полей
- **События:** Сохранение, отмена
- **Результат:** Новый пайщик в системе

### 2.2 Форма редактирования пайщика
- **Расположение:** Модальное окно при нажатии "Изменить"
- **Назначение:** Редактирование данных существующего пайщика
- **Поля:**
  - ФИО
  - Статус
  - Дата вступления
  - Контакт
  - Адрес
  - Примечания
- **Валидация:** Проверка корректности данных
- **События:** Сохранение, отмена
- **Результат:** Обновленные данные пайщика

### 2.3 Форма добавления паевого взноса
- **Расположение:** Модальное окно в разделе "Паевые взносы"
- **Назначение:** Добавление нового паевого взноса
- **Поля:**
  - Пайщик (выбор из списка)
  - Тип взноса (вступительный, паевой и др.)
  - Форма оплаты (наличные, безналичные, имущество)
  - Сумма (если не имущество)
  - Описание имущества (если форма оплаты "имущество")
  - Дата (по умолчанию сегодня)
  - Описание
  - Статус оплаты
  - Номер документа
- **Валидация:** Проверка обязательных полей, корректности суммы
- **События:** Сохранение, отмена
- **Результат:** Новый паевой взнос в системе

### 2.4 Форма оценки имущества
- **Расположение:** Модальное окно при нажатии "Оценить" для имущественного взноса
- **Назначение:** Проведение оценки имущества
- **Поля:**
  - Описание имущества (только для чтения)
  - Дата оценки
  - Оценочная стоимость
  - Метод оценки
  - Оценщик/Эксперт
  - Примечания к оценке
- **Валидация:** Проверка корректности суммы
- **События:** Сохранение, отмена
- **Результат:** Обновленная стоимость имущественного взноса

### 2.5 Форма добавления бухгалтерской проводки
- **Расположение:** Модальное окно в разделе "Бухгалтерия"
- **Назначение:** Добавление новой бухгалтерской проводки
- **Поля:**
  - Дата (по умолчанию сегодня)
  - Сумма
  - Дебетовый счет
  - Кредитовый счет
  - Описание
- **Валидация:** Проверка обязательных полей
- **События:** Сохранение, отмена
- **Результат:** Новая бухгалтерская проводка в системе

### 2.6 Форма загрузки документа
- **Расположение:** Модальное окно в разделе "Документы"
- **Назначение:** Загрузка нового документа
- **Поля:**
  - Файл (выбор файла)
  - Название документа
  - Тип документа
  - Описание
- **Валидация:** Проверка обязательных полей
- **События:** Загрузка, отмена
- **Результат:** Новый документ в системе

### 2.7 Форма обработки заявления
- **Расположение:** Модальное окно при обработке заявления
- **Назначение:** Принятие решения по заявлению на вступление
- **Поля:**
  - Информация о заявителе
  - Желаемый размер взноса
  - Форма оплаты
  - Решение (одобрить/отклонить)
  - Комментарии к решению
- **Валидация:** Проверка корректности решения
- **События:** Сохранение решения, отмена
- **Результат:** Обновленный статус заявления, новый пайщик при одобрении

### 2.8 Форма назначения заседания
- **Расположение:** Модальное окно в разделе "Заседания"
- **Назначение:** Назначение нового заседания
- **Поля:**
  - Дата заседания
  - Время заседания
  - Место проведения
  - Тип заседания
  - Тема заседания
  - Повестка дня
- **Валидация:** Проверка обязательных полей
- **События:** Создание заседания, отмена
- **Результат:** Новое заседание в системе

### 2.9 Форма проведения заседания
- **Расположение:** Модальное окно при нажатии "Провести"
- **Назначение:** Проведение заседания и фиксация решений
- **Поля:**
  - Список участников
  - Форма добавления участника
  - Список принятых решений
  - Форма добавления решения
- **Валидация:** Проверка корректности данных
- **События:** Добавление участника, добавление решения, завершение заседания
- **Результат:** Обновленный статус заседания, сформированный протокол

## 3. Боковые меню (40% ширины экрана)

### 3.1 Боковое меню просмотра пайщика
- **Расположение:** Правая сторона экрана (40% ширины)
- **Назначение:** Просмотр детальной информации о пайщике
- **Элементы:**
  - Личные данные (ID, ФИО, дата вступления, статус)
  - Контактная информация
  - Паевые взносы пайщика
  - Кнопки действий (редактировать, сформировать удостоверение)
- **Источник данных:** Объект пайщика из массива members

### 3.2 Боковое меню просмотра паевого взноса
- **Расположение:** Правая сторона экрана (40% ширины)
- **Назначение:** Просмотр детальной информации о паевом взносе
- **Элементы:**
  - Данные взноса (ID, пайщик, тип, сумма, дата)
  - Описание и статус
  - Кнопки действий (редактировать, удалить)
- **Источник данных:** Объект взноса из массива payments

### 3.3 Боковое меню просмотра бухгалтерской проводки
- **Расположение:** Правая сторона экрана (40% ширины)
- **Назначение:** Просмотр детальной информации о проводке
- **Элементы:**
  - Данные проводки (ID, дата, счета, сумма, описание)
  - Связанная информация (если есть)
  - Кнопки действий (редактировать, удалить)
- **Источник данных:** Объект проводки из массива transactions

### 3.4 Боковое меню просмотра документа
- **Расположение:** Правая сторона экрана (40% ширины)
- **Назначение:** Просмотр детальной информации о документе
- **Элементы:**
  - Данные документа (ID, название, тип, дата, размер)
  - Кнопки действий (открыть файл, скачать, удалить)
- **Источник данных:** Объект документа из массива documents

### 3.5 Боковое меню просмотра заявления
- **Расположение:** Правая сторона экрана (40% ширины)
- **Назначение:** Просмотр детальной информации о заявлении
- **Элементы:**
  - Личные данные заявителя
  - Дополнительная информация
  - Статус и дата подачи
  - Кнопки действий (обработать, закрыть)
- **Источник данных:** Объект заявления из массива applications

## 4. Формы отчетности

- **Расположение:** Раздел "Отчеты"
- **Назначение:** Выбор типа отчета для генерации
- **Элементы:**
  - Кнопки выбора типа отчета
  - Область вывода сгенерированного отчета
- **Типы отчетов:**
  - Отчет по пайщикам
  - Отчет по паевым взносам
  - Бухгалтерский баланс
  - Отчет о финансовых результатах

## 5. Специальные формы

### 5.1 Форма настройки директории C:\КООПЕРАНТ
- **Расположение:** Модальное окно при запуске системы
- **Назначение:** Настройка доступа к папке хранения данных
- **Элементы:**
  - Инструкции по созданию папки
  - Кнопка настройки доступа
- **Результат:** Настроен доступ к папке C:\КООПЕРАНТ

### 5.2 Форма заявления на вступление (отдельное окно)
- **Расположение:** Отдельное окно браузера
- **Назначение:** Самостоятельная подача заявления пайщиком
- **Поля:**
  - Личные данные (ФИО, дата рождения, паспорт)
  - Контактная информация
  - Род занятий и доход
  - Желаемый размер паевого взноса
  - Форма оплаты
  - Согласия и чекбоксы
- **Валидация:** Проверка обязательных полей и согласий
- **Результат:** Новое заявление в системе

## 7. Технические особенности

### 7.1 Использование File System API
- Сохранение данных в папке C:\КООПЕРАНТ
- Автоматическое сохранение при изменениях
- Резервное копирование данных
- Безопасность через согласие пользователя

### 7.2 Локальное хранение данных
- Все данные хранятся на устройстве пользователя
- Никакие данные не передаются на серверы
- Использование JSON формата для хранения
- Поддержка резервного механизма для других браузеров

## 8. Заключение

Все таблицы и формы системы разработаны с учетом удобства использования, безопасности данных и соответствия требованиям законодательства. Структура данных унифицирована, что обеспечивает целостность информации и простоту сопровождения системы. Боковые меню обеспечивают удобный просмотр детальной информации без сдвига основного интерфейса, а модальные окна используются для ввода данных и подтверждения операций.
