# Процесс настройки автоматического сохранения документов в папку C:\КООПЕРАНТ

## 1. Подготовка системы

### 1.1 Создание папки C:\КООПЕРАНТ
Перед началом использования системы необходимо создать папку C:\КООПЕРАНТ:
1. Откройте Проводник Windows
2. Перейдите на диск C:\
3. Щелкните правой кнопкой мыши по свободному месту
4. Выберите "Создать" → "Папка"
5. Назовите папку "КООПЕРАНТ"

### 1.2 Проверка прав доступа
Убедитесь, что у вас есть права на запись в папку C:\КООПЕРАНТ:
1. Щелкните правой кнопкой по папке C:\КООПЕРАНТ
2. Выберите "Свойства"
3. Перейдите на вкладку "Безопасность"
4. Убедитесь, что у вашей учетной записи есть права "Полный доступ"

## 2. Настройка доступа к файловой системе

### 2.1 Первый запуск системы
При первом запуске системы в браузере может потребоваться предоставить разрешение на доступ к файловой системе:
1. Откройте систему в поддерживаемом браузере (Chrome 86+, Edge 86+, Opera 72+)
2. При попытке сохранения данных браузер запросит разрешение на доступ к файловой системе
3. Нажмите "Разрешить" или "Позволить"

### 2.2 Настройка директории для документов
Для настройки автоматического сохранения документов:
1. Перейдите в раздел "Документы"
2. Нажмите кнопку "Настроить хранение"
3. В открывшемся окне выберите папку C:\КООПЕРАНТ
4. Подтвердите выбор папки

## 3. Автоматическое сохранение различных типов документов

### 3.1 Заявления на вступление
При подаче заявления на вступление:
- Заявление автоматически сохраняется в подпапке Applications
- Имя файла формируется по шаблону: application_[ID]_[ФИО_пайщика].json
- Файл содержит все данные заявления в формате JSON

### 3.2 Удостоверения пайщиков
При создании удостоверения для нового пайщика:
- Удостоверение автоматически сохраняется в подпапке Certificates
- Имя файла формируется по шаблону: certificate_[ID]_[ФИО_пайщика].json
- Файл содержит все данные удостоверения в формате JSON

### 3.3 Протоколы заседаний
При формировании протокола заседания:
- Протокол автоматически сохраняется в подпапке Protocols
- Имя файла формируется по шаблону: protocol_[ID]_[тема_заседания]_[дата].json
- Файл содержит все данные протокола в формате JSON

## 4. Структура папок

После настройки система автоматически создаст следующую структуру в папке C:\КООПЕРАНТ:

```
C:\КООПЕРАНТ\
├── Applications\          # Заявления на вступление
├── Certificates\          # Удостоверения пайщиков
├── Protocols\            # Протоколы заседаний
├── Contracts\            # Договоры
├── Reports\              # Отчеты
├── Payments\             # Платежные документы
├── Other\                # Прочие документы
└── backups\              # Резервные копии
```

## 5. Проверка автоматического сохранения

### 5.1 Проверка сохранения заявлений
1. Подайте новое заявление на вступление
2. Перейдите в папку C:\КООПЕРАНТ\Applications
3. Убедитесь, что появился новый файл с данными заявления

### 5.2 Проверка сохранения удостоверений
1. Примите нового пайщика (автоматически создастся удостоверение)
2. Перейдите в папку C:\КООПЕРАНТ\Certificates
3. Убедитесь, что появился новый файл с данными удостоверения

### 5.3 Проверка сохранения протоколов
1. Создайте новое заседание и сформируйте протокол
2. Перейдите в папку C:\КООПЕРАНТ\Protocols
3. Убедитесь, что появился новый файл с данными протокола

## 6. Резервное копирование

### 6.1 Автоматическое резервное копирование
Система автоматически создает резервные копии важных документов:
- Резервные копии создаются ежедневно
- Копии хранятся в подпапке backups
- Имена файлов включают дату создания копии

### 6.2 Восстановление из резервной копии
Для восстановления данных из резервной копии:
1. Перейдите в папку C:\КООПЕРАНТ\backups
2. Найдите файл резервной копии за нужную дату
3. Скопируйте файл в корень папки C:\КООПЕРАНТ
4. Переименуйте файл в coop_data.json
5. Перезапустите систему

## 7. Безопасность данных

### 7.1 Защита персональных данных
- Все персональные данные хранятся локально на вашем компьютере
- Система не передает данные на внешние серверы
- Доступ к данным контролируется пользователем

### 7.2 Шифрование (опционально)
Для дополнительной защиты данных рекомендуется:
- Использовать шифрование диска (BitLocker для Windows)
- Установить пароль на учетную запись пользователя
- Регулярно обновлять операционную систему

## 8. Устранение неполадок

### 8.1 Проблемы с доступом к папке
Если система не может сохранить документы:
1. Проверьте, существует ли папка C:\КООПЕРАНТ
2. Проверьте права доступа к папке
3. Убедитесь, что антивирус не блокирует доступ
4. Попробуйте перезапустить браузер

### 8.2 Проблемы с File System API
Если используется браузер без поддержки File System API:
- Система будет использовать localStorage с ограничениями
- Для полной функциональности используйте поддерживаемый браузер
- Рекомендуется Chrome, Edge или Opera последних версий

## 9. Рекомендации по использованию

### 9.1 Регулярные проверки
- Периодически проверяйте наличие файлов в подпапках
- Убедитесь, что автоматическое сохранение работает корректно
- Проверяйте целостность данных

### 9.2 Обслуживание
- Регулярно очищайте старые резервные копии
- Архивируйте старые документы
- Обновляйте систему по мере выхода новых версий

## 10. Поддержка

Если у вас возникли проблемы с настройкой автоматического сохранения:
1. Проверьте файл error_guide.md для решения типичных проблем
2. Убедитесь, что используете поддерживаемый браузер
3. Проверьте права доступа к папке C:\КООПЕРАНТ
4. При необходимости обратитесь к администратору системы
