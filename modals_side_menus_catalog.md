# Каталог модальных окон и боковых меню системы

## 1. Общая информация

### 1.1 Назначение модальных окон
Модальные оконца используются для:
- Ввода данных в формах
- Подтверждения операций
- Просмотра детальной информации
- Отображения уведомлений

### 1.2 Назначение боковых меню
Боковые меню (40% ширины экрана) используются для:
- Просмотра детальной информации
- Отображения данных без перекрытия основного интерфейса
- Предоставления дополнительных действий
- Удобного просмотра записей

## 2. Модальные окна системы

### 2.1 Модальное окно добавления пайщика
- **Вызов:** Кнопка "Добавить пайщика" в разделе "Пайщики"
- **Назначение:** Добавление нового пайщика в систему
- **Содержание:**
  - Форма с полями: ФИО, статус, дата вступления, контакт, адрес, примечания
  - Кнопки: Сохранить, Отмена
  - Валидация обязательных полей
- **Результат:** Новый пайщик в системе

### 2.2 Модальное окно редактирования пайщика
- **Вызов:** Кнопка "Изменить" в строке пайщика
- **Назначение:** Редактирование данных существующего пайщика
- **Содержание:**
  - Форма с заполненными текущими данными
  - Поля для редактирования: ФИО, статус, дата вступления, контакт, адрес, примечания
  - Кнопки: Сохранить, Отмена
- **Результат:** Обновленные данные пайщика

### 2.3 Модальное окно добавления паевого взноса
- **Вызов:** Кнопка "Добавить взнос" в разделе "Паевые взносы"
- **Назначение:** Добавление нового паевого взноса
- **Содержание:**
  - Выбор пайщика из списка
  - Выбор типа взноса
  - Выбор формы оплаты (наличные, безналичные, имущество)
  - Поля для суммы или описания имущества
  - Дата и описание взноса
  - Кнопки: Сохранить, Отмена
- **Особенности:** Условное отображение полей в зависимости от формы оплаты
- **Результат:** Новый паевой взнос в системе

### 2.4 Модальное окно редактирования паевого взноса
- **Вызов:** Кнопка "Изменить" в строке паевого взноса
- **Назначение:** Редактирование существующего паевого взноса
- **Содержание:**
  - Форма с текущими данными взноса
  - Возможность изменения всех параметров
  - Кнопки: Сохранить, Отмена
- **Результат:** Обновленный паевой взнос

### 2.5 Модальное окно оценки имущества
- **Вызов:** Кнопка "Оценить" для неоцененного имущественного взноса
- **Назначение:** Проведение оценки имущества
- **Содержание:**
  - Описание имущества (только для чтения)
  - Дата оценки
  - Оценочная стоимость
  - Метод оценки
  - Оценщик/Эксперт
  - Примечания к оценке
  - Кнопки: Оформить акт оценки, Отмена
- **Результат:** Обновленная стоимость имущественного взноса

### 2.6 Модальное окно добавления бухгалтерской проводки
- **Вызов:** Кнопка "Добавить проводку" в разделе "Бухгалтерия"
- **Назначение:** Добавление новой бухгалтерской проводки
- **Содержание:**
  - Дата проводки
  - Сумма
  - Дебетовый счет
  - Кредитовый счет
  - Описание
  - Кнопки: Сохранить, Отмена
- **Результат:** Новая бухгалтерская проводка

### 2.7 Модальное окно редактирования проводки
- **Вызов:** Кнопка "Изменить" в строке проводки
- **Назначение:** Редактирование существующей проводки
- **Содержание:**
  - Форма с текущими данными проводки
  - Поля для редактирования
  - Кнопки: Сохранить, Отмена
- **Результат:** Обновленная бухгалтерская проводка

### 2.8 Модальное окно загрузки документа
- **Вызов:** Кнопка "Загрузить документ" в разделе "Документы"
- **Назначение:** Загрузка нового документа
- **Содержание:**
  - Поле выбора файла
  - Название документа
  - Тип документа
  - Описание
  - Кнопки: Загрузить, Отмена
- **Результат:** Новый документ в системе

### 2.9 Модальное окно обработки заявления
- **Вызов:** Кнопка "Обработать" в строке заявления
- **Назначение:** Принятие решения по заявлению на вступление
- **Содержание:**
  - Информация о заявителе
  - Желаемый размер взноса
  - Форма оплаты
  - Выбор решения (одобрить/отклонить)
  - Комментарии к решению
  - Кнопки: Принять решение, Отмена
- **Результат:** Обновленный статус заявления, новый пайщик при одобрении

### 2.10 Модальное окно назначения заседания
- **Вызов:** Кнопка "Назначить заседание" в разделе "Заседания"
- **Назначение:** Создание нового заседания
- **Содержание:**
  - Дата и время заседания
  - Место проведения
  - Тип заседания
  - Тема и повестка дня
  - Кнопки: Создать заседание, Отмена
- **Результат:** Новое запланированное заседание

### 2.11 Модальное окно проведения заседания
- **Вызов:** Кнопка "Провести" в строке заседания
- **Назначение:** Проведение заседания и фиксация решений
- **Содержание:**
  - Информация о заседании
  - Список участников
  - Форма добавления участника
  - Список решений
  - Форма добавления решения
  - Кнопки: Завершить заседание, Отмена
- **Результат:** Проведенное заседание с протоколом

### 2.12 Модальное окно настройки директории C:\КООПЕРАНТ
- **Вызов:** При первом запуске системы или при необходимости настройки
- **Назначение:** Настройка доступа к папке хранения данных
- **Содержание:**
  - Инструкции по созданию папки
  - Кнопка настройки доступа
  - Кнопки: Настроить, Отмена
- **Результат:** Настроен доступ к папке C:\КООПЕРАНТ

### 2.13 Модальное окно просмотра протокола заседания
- **Вызов:** Кнопка "Протокол" в строке заседания
- **Назначение:** Просмотр детального протокола заседания
- **Содержание:**
  - Информация о заседании
  - Список участников
  - Принятые решения
  - Кнопки: Сохранить как PDF, Закрыть
- **Результат:** Просмотр протокола

### 2.14 Модальное окно генерации удостоверения
- **Вызов:** Кнопка "Сформировать удостоверение" в карточке пайщика
- **Назначение:** Генерация удостоверения пайщика
- **Содержание:**
  - Информация о пайщике
  - Данные удостоверения
  - Кнопки: Сохранить как PDF, Закрыть
- **Результат:** Сформированное удостоверение

## 3. Боковые меню системы (40% ширины экрана)

### 3.1 Боковое меню просмотра пайщика
- **Вызов:** Кнопка "Просмотр" в строке пайщика
- **Назначение:** Просмотр детальной информации о пайщике
- **Расположение:** Правая сторона экрана (40% ширины)
- **Содержание:**
  - Личные данные пайщика
  - Контактная информация
  - Статус и дата вступления
  - Паевые взносы пайщика
  - Кнопки действий: Редактировать, Сформировать удостоверение, Закрыть
- **Особенности:** Перекрывает основной интерфейс, но не сдвигает его
- **Результат:** Просмотр детальной информации

### 3.2 Боковое меню просмотра паевого взноса
- **Вызов:** Кнопка "Просмотр" в строке паевого взноса
- **Назначение:** Просмотр деталей паевого взноса
- **Расположение:** Правая сторона экрана (40% ширины)
- **Содержание:**
  - Данные взноса (ID, пайщик, тип, сумма, дата)
  - Описание и статус оплаты
  - Кнопки действий: Редактировать, Удалить, Закрыть
- **Результат:** Просмотр детальной информации о взносе

### 3.3 Боковое меню просмотра бухгалтерской проводки
- **Вызов:** Кнопка "Просмотр" в строке бухгалтерской проводки
- **Назначение:** Просмотр деталей бухгалтерской проводки
- **Расположение:** Правая сторона экрана (40% ширины)
- **Содержание:**
  - Данные проводки (ID, дата, счета, сумма, описание)
  - Связанная информация (если есть)
  - Кнопки действий: Редактировать, Удалить, Закрыть
- **Результат:** Просмотр детальной информации о проводке

### 3.4 Боковое меню просмотра документа
- **Вызов:** Кнопка "Просмотр" в строке документа
- **Назначение:** Просмотр информации о документе
- **Расположение:** Правая сторона экрана (40% ширины)
- **Содержание:**
  - Данные документа (ID, название, тип, дата, размер)
  - Кнопки действий: Открыть файл, Скачать, Удалить, Закрыть
- **Результат:** Просмотр информации о документе

### 3.5 Боковое меню просмотра заявления
- **Вызов:** Кнопка "Просмотр" в строке заявления
- **Назначение:** Просмотр детальной информации о заявлении
- **Расположение:** Правая сторона экрана (40% ширины)
- **Содержание:**
  - Личные данные заявителя
  - Дополнительная информация
  - Статус и дата подачи
  - Кнопки действий: Обработать, Закрыть
- **Результат:** Просмотр информации о заявлении

## 4. Технические особенности реализации

### 4.1 Структура модального окна
```html
<div id="modal" class="modal" style="display: none;">
    <div class="modal-content">
        <span class="close" onclick="closeModal()">&times;</span>
        <div id="modal-body"></div>
    </div>
</div>
```

### 4.2 Структура бокового меню
```html
<div id="side-menu" class="side-menu">
    <div class="side-menu-header">
        <h3 id="side-menu-title">Детали</h3>
        <span class="close" onclick="closeSideMenu()">&times;</span>
    </div>
    <div id="side-menu-body" class="side-menu-body">
        <!-- Содержимое бокового меню -->
    </div>
</div>
```

### 4.3 Функции управления модальными окнами
```javascript
// Показ модального окна
function showModal(content) {
    document.getElementById('modal-body').innerHTML = content;
    document.getElementById('modal').style.display = 'block';
}

// Закрытие модального окна
function closeModal() {
    document.getElementById('modal').style.display = 'none';
    document.getElementById('modal-body').innerHTML = '';
}
```

### 4.4 Функции управления боковыми меню
```javascript
// Показ бокового меню
function showSideMenu(title, content) {
    const sideMenu = document.getElementById('side-menu');
    const overlay = document.createElement('div');
    overlay.className = 'side-menu-overlay';
    overlay.classList.add('active');
    overlay.onclick = closeSideMenu;
    document.body.appendChild(overlay);
    
    document.getElementById('side-menu-title').textContent = title;
    document.getElementById('side-menu-body').innerHTML = content;
    sideMenu.classList.add('active');
}

// Закрытие бокового меню
function closeSideMenu() {
    const sideMenu = document.getElementById('side-menu');
    const overlay = document.querySelector('.side-menu-overlay');
    
    sideMenu.classList.remove('active');
    
    if (overlay) {
        document.body.removeChild(overlay);
    }
}
```

## 5. Правила использования интерфейсов

### 5.1 Модальные окна
- Используются для ввода данных и подтверждения операций
- Имеют четкие заголовки и кнопки действий
- Содержат валидацию данных при необходимости
- Закрываются по кнопке "Отмена" или крестику

### 5.2 Боковые меню
- Используются для просмотра информации
- Занимают 40% ширины экрана
- Перекрывают основной интерфейс (не сдвигают его)
- Имеют кнопку закрытия в заголовке
- Закрываются при клике на оверлей

## 6. Доступность и удобство использования

### 6.1 Клавиатурная навигация
- Закрытие окон по клавише ESC
- Фокусировка на элементах управления
- Последовательная навигация по Tab

### 6.2 Адаптивность
- Корректное отображение на разных размерах экрана
- Адаптация под мобильные устройства
- Сохранение функциональности на всех устройствах

## 7. Безопасность интерфейсов

### 7.1 Защита данных
- Все данные отображаются с учетом прав доступа
- Конфиденциальная информация защищена
- Нет утечки данных через интерфейс

### 7.2 Валидация
- Проверка данных на вводе
- Защита от XSS-атак
- Очистка пользовательского ввода

## 8. Заключение

Модальные окна и боковые меню системы обеспечивают удобное и эффективное взаимодействие с информацией. Боковые меню с 40% шириной экрана обеспечивают удобный просмотр детальной информации без перекрытия основного интерфейса, в то время как модальные окна используются для ввода данных и подтверждения операций. Все интерфейсы разработаны с учетом принципов удобства использования и безопасности данных.
