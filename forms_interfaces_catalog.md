# Каталог форм и интерфейсов информационной системы учета пайщиков кооператива

## 1. Общая информация

### 1.1 Назначение интерфейсов
Интерфейсы системы обеспечивают удобное взаимодействие пользователей с системой, позволяя эффективно выполнять все необходимые операции по учету пайщиков, паевых взносов, бухгалтерскому учету и документообороту.

### 1.2 Принципы проектирования интерфейсов
- Простота и интуитивность использования
- Единообразие элементов управления
- Адаптивность под разные размеры экрана
- Быстрый доступ к часто используемым функциям
- Четкая навигация между разделами

## 2. Основные интерфейсы системы

### 2.1 Главная страница (dashboard)
- **Расположение:** index.html, раздел с ID "dashboard"
- **Назначение:** Общая статистика и навигация по системе
- **Элементы:**
  - Заголовок системы
  - Навигационное меню
  - Карточки статистики:
    - Всего пайщиков
    - Активных пайщиков
    - Паевых взносов
    - Задолженность
  - Краткое описание системы
- **Функции:** Навигация по разделам, отображение общей статистики

### 2.2 Интерфейс учета пайщиков (members)
- **Расположение:** index.html, раздел с ID "members"
- **Назначение:** Управление информацией о пайщиках
- **Элементы:**

## 3. Формы ввода данных

### 3.1 Форма добавления пайщика
- **Тип:** Модальное окно
- **Расположение:** Вызывается из раздела "Пайщики"
- **Поля:**
  - ФИО (обязательное)
  - Статус (выбор из списка)
  - Дата вступления (календарь)
  - Контакт (телефон/Email)
  - Адрес (текстовое поле)
  - Примечания (многострочное текстовое поле)
- **Кнопки:** Сохранить, Отмена
- **Валидация:** Проверка обязательных полей

### 3.2 Форма редактирования пайщика
- **Тип:** Модальное окно
- **Расположение:** Вызывается из таблицы пайщиков
- **Поля:** Те же, что и в форме добавления, с предзаполненными значениями
- **Кнопки:** Сохранить, Отмена
- **Валидация:** Проверка корректности данных

### 3.3 Форма добавления паевого взноса
- **Тип:** Модальное окно
- **Расположение:** Вызывается из раздела "Паевые взносы"
- **Поля:**
  - Выбор пайщика (выпадающий список)
  - Тип взноса (выбор из списка)
  - Форма оплаты (выбор из списка)
  - Сумма (числовое поле, скрывается при выборе "имущество")
  - Описание имущества (текстовое поле, появляется при выборе "имущество")
  - Дата (календарь)
  - Описание (многострочное текстовое поле)
  - Статус оплаты (чекбокс)
  - Номер документа (текстовое поле)
- **Кнопки:** Сохранить, Отмена
- **Валидация:** Проверка обязательных полей, корректности суммы

### 3.4 Форма редактирования паевого взноса
- **Тип:** Модальное окно
- **Расположение:** Вызывается из таблицы взносов
- **Поля:** Те же, что и в форме добавления, с предзаполненными значениями
- **Кнопки:** Сохранить, Отмена
- **Валидация:** Проверка корректности данных

### 3.5 Форма добавления бухгалтерской проводки
- **Тип:** Модальное окно
- **Расположение:** Вызывается из раздела "Бухгалтерия"
- **Поля:**
  - Дата (календарь)
  - Сумма (числовое поле)
  - Дебетовый счет (текстовое поле)
  - Кредитовый счет (текстовое поле)
  - Описание (многострочное текстовое поле)
- **Кнопки:** Сохранить, Отмена
- **Валидация:** Проверка обязательных полей, корректности суммы и счетов

### 3.6 Форма загрузки документа
- **Тип:** Модальное окно
- **Расположение:** Вызывается из раздела "Документы"
- **Поля:**
  - Выбор файла (input type="file")
  - Название документа (текстовое поле)
  - Тип документа (выбор из списка)
  - Описание (многострочное текстовое поле)
- **Кнопки:** Загрузить, Отмена
- **Валидация:** Проверка обязательных полей, формата файла

### 3.7 Форма назначения заседания
- **Тип:** Модальное окно
- **Расположение:** Вызывается из раздела "Заседания"
- **Поля:**
  - Дата (календарь)
  - Время (время)
  - Место проведения (текстовое поле)
  - Тип заседания (выбор из списка)
  - Тема (текстовое поле)
  - Повестка дня (многострочное текстовое поле)
- **Кнопки:** Создать заседание, Отмена
- **Валидация:** Проверка обязательных полей

### 3.8 Форма проведения заседания
- **Тип:** Модальное окно
- **Расположение:** Вызывается из таблицы заседаний
- **Элементы:**
  - Информация о заседании (только для чтения)
  - Список участников с возможностью добавления
  - Список решений с возможностью добавления
  - Кнопки: Завершить заседание, Отмена
- **Валидация:** Проверка корректности данных

## 5. Модальные окна

### 5.1 Модальное окно настройки директории
- **Назначение:** Настройка доступа к папке C:\КООПЕРАНТ
- **Содержание:**
  - Инструкции по созданию папки
  - Кнопка настройки доступа
  - Информация о процессе
- **Кнопки:** Настроить доступ, Отмена
- **Условия показа:** При первом запуске системы

### 5.2 Модальное окно оценки имущества
- **Назначение:** Проведение оценки имущественного взноса
- **Поля:**
  - Описание имущества (только для чтения)
  - Дата оценки (календарь)
  - Оценочная стоимость (числовое поле)
  - Метод оценки (выбор из списка)
  - Оценщик/Эксперт (текстовое поле)
  - Примечания к оценке (многострочное текстовое поле)
- **Кнопки:** Оформить акт оценки, Отмена
- **Условия показа:** При нажатии "Оценить" для неоцененного имущественного взноса

### 5.3 Модальное окно обработки заявления
- **Назначение:** Обработка заявления на вступление
- **Содержание:**
  - Информация о заявителе
  - Желаемый размер взноса
  - Форма оплаты
- **Поля:**
  - Решение (одобрить/отклонить)
  - Комментарии к решению (многострочное текстовое поле)
- **Кнопки:** Принять решение, Отмена
- **Условия показа:** При обработке заявления

## 6. Форма подачи заявления (отдельное окно)

Форма для самостоятельной подачи заявления на вступление в кооператив.

- **Личные данные:**
  - Фамилия (обязательное)

  - Форма оплаты (обязательное)
  - Описание имущества (если форма оплаты "имущество")
  - Дополнительная информация

- **Согласия:**
  - Согласие с уставом (обязательное)
  - Согласие на обработку персональных данных (обязательное)

### 6.3 Кнопки
- Подать заявление
- Отмена

### 6.4 Валидация

## 10. Заключение

Все интерфейсы системы спроектированы с учетом принципов удобства использования, безопасности и эффективности. Боковые меню обеспечивают удобный просмотр детальной информации, не сдвигая основной интерфейс, а модальные окна используются для форм ввода данных. Система обеспечивает полный цикл работы с информацией пайщиков и паевыми взносами, включая все необходимые элементы управления и валидации.
