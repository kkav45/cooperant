# Новые функции системы учета пайщиков кооператива

## 1. Заявления на вступление

### 1.1 Подача заявления
- Пайщик может самостоятельно заполнить форму заявления на вступление в кооператив
- Форма включает все необходимые поля: личные данные, контактная информация, размер и форма оплаты паевого взноса
- После заполнения заявление автоматически сохраняется в папке C:\КООПЕРАНТ\Applications

### 1.2 Обработка заявления
- Администратор может просматривать поступившие заявления
- Возможность массового рассмотрения заявлений
- Принятие решения об одобрении или отказе во вступлении
- При положительном решении автоматически создается запись о новом пайщике

### 1.3 Ожидаемые взносы
- При подаче заявления автоматически создается запись об ожидаемом паевом взносе
- Это позволяет отслеживать обязательства пайщиков до их официального вступления
- Ожидаемые взносы отображаются отдельно в системе

## 2. Удостоверения пайщиков

### 2.1 Автоматическое создание удостоверения
- При принятии пайщика в кооператив автоматически формируется удостоверение
- Удостоверение содержит ФИО пайщика, дату выдачи и уникальный номер
- Удостоверение автоматически сохраняется в папке C:\КООПЕРАНТ\Certificates

### 2.2 Просмотр и печать
- Возможность просмотра удостоверения в удобном формате
- Функция экспорта удостоверения в формате PDF
- Управление статусом удостоверения

## 3. Заседания и протоколы

### 3.1 Планирование заседаний
- Функция назначения заседаний правления, общих собраний и комитетов
- Указание даты, времени, места и повестки дня
- Возможность массового рассмотрения заявлений на вступление на заседании

### 3.2 Проведение заседаний
- Формирование списка участников заседания
- Фиксация принятых решений
- Назначение ответственных за исполнение решений
- Установка сроков исполнения

### 3.3 Протоколы заседаний
- Автоматическое формирование протокола заседания
- Протокол содержит информацию об участниках, повестке дня и принятых решениях
- Протокол автоматически сохраняется в папке C:\КООПЕРАНТ\Protocols
- Функция экспорта протокола в формате PDF

## 4. Автоматическое сохранение документов

### 4.1 Структура папок
Система автоматически создает следующую структуру в папке C:\КООПЕРАНТ:
- Applications - заявления на вступление
- Certificates - удостоверения пайщиков
- Protocols - протоколы заседаний
- Contracts - договоры
- Reports - отчеты
- Payments - платежные документы
- Other - прочие документы

### 4.2 Безопасность данных
- Все документы сохраняются локально на компьютере пользователя
- Доступ к документам контролируется пользователем
- Система не передает документы на внешние серверы

## 5. Интеграция с процессом вступления

### 5.1 Полный цикл вступления
1. Пайщик заполняет форму заявления
2. Заявление сохраняется в системе и в папке C:\КООПЕРАНТ\Applications
3. Администратор рассматривает заявление
4. При положительном решении создается запись о новом пайщике
5. Автоматически формируется удостоверение и сохраняется в C:\КООПЕРАНТ\Certificates
6. При необходимости назначается заседание для рассмотрения нескольких заявлений
7. Протокол заседания сохраняется в C:\КООПЕРАНТ\Protocols

### 5.2 Контроль процесса
- Система позволяет отслеживать статус каждого заявления
- Возможность массовой обработки заявлений
- Автоматическое формирование отчетности по процессу вступления
