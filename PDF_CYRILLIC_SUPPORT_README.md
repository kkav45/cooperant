# Информационная система учета пайщиков кооператива

## Решение проблемы с кодировкой в PDF

### Проблема
Ранее при экспорте документов в PDF русский текст отображался некорректно из-за отсутствия поддержки кириллических символов в стандартных шрифтах jsPDF.

### Решение
Мы реализовали поддержку кириллических шрифтов в PDF-экспорте следующим образом:

1. **Создан специальный файл шрифта** - `fonts/roboto-cyrillic-normal.js` с полной таблицей метрик для кириллических символов
2. **Обновлены все функции экспорта** - все функции в `pdf_export.js` теперь используют кириллический шрифт
3. **Автоматическая интеграция** - файл подключается в `index.html`, поэтому все функции доступны в приложении

### Файлы
- `fonts/roboto-cyrillic-normal.js` - файл шрифта с поддержкой кириллицы
- `pdf_export.js` - функции экспорта с поддержкой кириллицы
- `app.js` - основное приложение (без дублирования функций экспорта)
- `index.html` - главная страница с подключением всех необходимых файлов

### Поддерживаемые функции экспорта
- `exportCertificateAsPDF(certificateId)` - экспорт удостоверений
- `exportMemberCertificateAsPDF(memberId)` - экспорт удостоверений пайщиков
- `exportMeetingProtocolAsPDF(meetingId)` - экспорт протоколов заседаний
- `exportReportAsPDF(reportTitle, reportContent)` - экспорт отчетов
- `exportApplicationAsPDF(applicationId)` - экспорт заявлений

### Особенности реализации
- Все функции автоматически загружают кириллический шрифт перед созданием PDF
- Никаких изменений в вызовах функций не требуется
- Русский текст теперь корректно отображается во всех экспортируемых PDF-документах

### Тестирование
